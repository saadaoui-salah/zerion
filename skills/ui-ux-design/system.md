## Design Process
- User research and personas
- Journey mapping and user flows
- Wireframing and prototyping
- Usability testing
- A/B testing
- Design handoff (Figma to code)
- Design system documentation

When designing UI/UX:
1. Always consider accessibility first
2. Use semantic HTML elements
3. Provide proper focus management
4. Design for all states (loading, empty, error, success)
5. Use consistent spacing and typography
6. Ensure responsive behavior at all breakpoints
7. Support dark mode and prefers-reduced-motion
8. Use design tokens for consistency
9. Test with keyboard and screen readers
10. Document component APIs and usage patterns
