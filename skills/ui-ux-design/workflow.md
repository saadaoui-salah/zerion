# UI/UX Design Workflow

## 1. Research
- Understand the target users and their goals
- Review existing design patterns and competitors
- Check current accessibility compliance
- Identify brand guidelines and constraints
- Review existing codebase for patterns

## 2. Wireframe
- Create low-fidelity layouts
- Define information architecture
- Map user flows and navigation
- Identify key interaction points
- Plan responsive breakpoints

## 3. Design
- Establish design tokens (colors, typography, spacing)
- Create component specifications
- Define states (default, hover, focus, active, disabled, loading, error)
- Plan animation and transitions
- Ensure accessibility compliance (contrast, focus, semantics)

## 4. Prototype
- Build interactive prototypes
- Test user flows
- Validate responsive behavior
- Check keyboard navigation
- Test with screen readers

## 5. Implement
- Use semantic HTML elements
- Apply design tokens consistently
- Implement responsive layouts
- Add animations and transitions
- Handle all component states
- Ensure keyboard accessibility

## 6. Test
- Validate WCAG compliance
- Test with screen readers (NVDA, VoiceOver)
- Test keyboard navigation
- Test responsive at all breakpoints
- Test dark mode
- Test reduced motion preferences
- Run Lighthouse audit
