# Python Development Workflow

## 1. Inspect
- Read the relevant Python files
- Check the project structure
- Review requirements.txt or pyproject.toml
- Identify the Python version and dependencies

## 2. Analyze
- Understand the code's purpose
- Identify design patterns used
- Check for type hints and docstrings
- Review error handling
- Look for potential performance issues

## 3. Plan
- Design the solution following Python best practices
- Consider edge cases and error scenarios
- Plan for testability
- Document the approach

## 4. Implement
- Write clean, idiomatic Python code
- Add proper type hints
- Include docstrings
- Handle exceptions appropriately
- Follow PEP 8 style

## 5. Test
- Write unit tests with pytest
- Add edge case tests
- Verify error handling
- Check type consistency

## 6. Verify
- Run the test suite
- Check for linting issues
- Verify type checking with mypy
- Review for security vulnerabilities
